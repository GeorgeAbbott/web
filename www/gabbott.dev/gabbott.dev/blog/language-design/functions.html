Unary Operators

There are, across languages, very few unary operators. The major ones are:

  1. Unary minus, to negate a value -
  2. Unary plus, to... convert a lambda to a function pointer?
  3. Increment ++ and decrement --
  4. Negation, i.e. not and bitwise negation ~
  5. The addressof operator, i.e. &
  6. The pointer dereference operator, i.e. *, in C
  7. sizeof

When designing a language, instead of adding special parsing for unary operators, we could get rid of all of these using a function instead.

For instance:

  • Unary minus: instead of -1, do negate(1)
  • Unary plus: get rid of it. Even in C, it only does conversions, and we shouldn't be doing conversions in this way.
  • Increment, decrement: Use preincrement(10), postdecrement(9), etc.
  • Addressof: either addrof function (like C++ std::addressof, or & as a function name (see Identifiers, potentially with Implicit Brackets
  • Pointer deference: same as with addressof. Either deref function, or * as a function name.

All in all, it seems a good idea to strike unary operators from the core language altogether.
