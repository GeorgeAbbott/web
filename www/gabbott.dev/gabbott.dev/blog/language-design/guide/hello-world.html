Hello, World! in the Close programming language

Jumping straight into it, here is the code for Hello World:


main: fn: i32
    print "Hello, World!"
end

This looks rather clean, but benefits from a few hidden features which I shall go over here. In general, Close strives (will strive) to not have much in the way of hidden gotchas, but some convenience things are provided, detailled below.

Firstly, you can notice the syntax is identifier: keyword. This use of the identifier first, before a type name, etc. is consistent throughout the language.

Secondly, the fn keyword is used to introduce a function. It consists of brackets after the fn, which can be omitted if there are no parameters. If they are, they are in name: type format. For instance, an alternative way of writing the main function is: 

main: fn(args: []str): i32

With the args parameter. The return type is given after another colon.

The print function is declared in std.io, but is re-exported in std.prelude, a special module that is included in every file, unless do not import std.prelude is given. print takes a string which can contain format specifiers in {}, and then any amount of further parameters (variadic arguments). For instance: 


print "Hello, {}, and goodbye {}.", "Tom", "Mike"
